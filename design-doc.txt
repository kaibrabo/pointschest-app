# PointsChest - Design Document

## Project Overview
Purpose: Simple, fast credit card comparison and discovery app

Description: PointsChest is a mobile app that helps users find the best credit card for their spending habits. Users browse a curated list of credit cards, search by category (dining, travel, gas, etc.) to find the highest reward rates, and tap "Apply Now" to go directly to the card issuer. Users can save favorite cards to a personal wallet and view their profile with a summary of their cards. Future phases will add user authentication, persistent storage, real-time card data, and a card recommendation engine.

## Requirements

### Technical Stack
- React Native (Expo SDK ~52)
- Expo Router (file-based routing)
- TypeScript
- Node.js / npm

### Environment & Setup
- No environment variables required (Phase 1 — static data)
- No database setup (local state only)
- No authentication setup (Phase 1)
- No storage configuration (Phase 1)

## Build Guidelines

### Development Process
1. Install dependencies: `npm install`
2. Start dev server: `npx expo start`
3. Run on iOS: `npx expo start --ios`
4. Run on Android: `npx expo start --android`
5. Run tests: `npm test`
6. Lint: `npm run lint`

### Folder Structure
- `app/` — Expo Router screens (entry points)
- `src/screens/` — Full screen components
- `src/data/` — Static card data
- `src/types/` — Shared TypeScript interfaces
- `assets/` — Fonts and images

### Error Handling & Logging
- Validate URLs before calling Linking.openURL (https:// scheme only)
- Handle empty/no-match search states gracefully
- Guard against missing optional fields (welcomeBonus, imageUrl)

## Notes
- All card data is currently static in src/data/creditCards.ts — 30 cards total
- Favorites are in-memory only (lost on app restart) until AsyncStorage is added
- The app-example/ folder is Expo boilerplate and should be deleted before launch
- google-services.json and GoogleService-Info.plist were generated but never integrated — delete them
- "Apply Now" links go directly to card issuer sites — ensure https:// validation before opening
- Card data accuracy: APR, bonuses, and benefits change frequently — add a "rates as of [date]" disclaimer
- Android keyboard handling uses keyboardDidShow/Hide (not keyboardWillShow/Hide like iOS) — needs fix
